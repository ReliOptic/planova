# Planova — Premortem Analysis

> 6개월 후 Planova가 실패했다고 가정하고, 그 원인을 지금 찾는다.

## 시나리오: "2026년 10월, Planova MAU 50명 미만으로 프로젝트 중단"

---

### Death Cause 1: "편차 숫자가 스트레스를 준다"

**살인 동기:** 매일 "+23분 초과" 같은 숫자를 보면 자괴감이 든다.
사용자는 "체중계"가 아니라 "체중계를 안 보는 선택"을 한다.

**확률:** 높음 (40%)

**예방 조치:**
- 편차를 **비난이 아닌 패턴**으로 프레이밍: "이번 주 보고서 작성은 평균 +18분. 다음번엔 1시간 20분으로 계획해볼까요?"
- 숫자만 보여주지 말고 **개선 추이**를 함께 보여줌: "지난주 정확도 71% → 이번 주 82%"
- 절대 판단 언어 사용 금지 ("나쁨", "실패", "비효율")
- **Streak Freeze**: 하루 안 해도 streak 유지 → 완벽주의 압박 제거

**사전 테스트:** 초기 베타에서 "스트레스 받는다" 피드백 비율 모니터링. 20% 초과 시 톤 조정.

---

### Death Cause 2: "투두앱이랑 뭐가 다른지 모르겠다"

**살인 동기:** 랜딩페이지에서 "시간 편차 가시화"를 강조했지만,
첫 사용 시 보이는 건 그냥 투두 + 타임라인. 차별화 기능(편차, Daily Pulse)을
경험하려면 작업을 여러 개 완료해야 함. 그 전에 이탈.

**확률:** 높음 (35%)

**예방 조치:**
- **첫 작업 완료 시 바로 편차 표시** — 10개 쌓일 때까지 안 기다림
- 온보딩에 샘플 데이터 제공: "이런 식으로 보입니다" 프리뷰
- Yesterday Ghosts로 빈 타임라인 방지 — 첫날부터 뭔가 보임
- 랜딩페이지에 실제 Daily Pulse 스크린샷 포함

**사전 테스트:** 신규 사용자의 "첫 편차 표시까지 걸리는 시간" 측정. 5분 이내 목표.

---

### Death Cause 3: "입력이 여전히 귀찮다"

**살인 동기:** Quick Add를 만들었지만, 자연어 파서가 한국어를 잘 못 알아먹거나
예상과 다르게 파싱되어 사용자가 결국 모달로 돌아감.
마찰 감소 효과 없음.

**확률:** 중간 (25%)

**예방 조치:**
- 파서를 과도하게 복잡하게 만들지 않음 — `"제목 시간"` 2-토큰만 확실히 처리
- 파싱 결과 실시간 프리뷰: 입력하면서 아래에 "제목: 회의 | 시간: 30분" 표시
- 파싱 실패 시 graceful fallback: 전체를 제목으로, duration 기본값
- 모달은 항상 접근 가능 — Quick Add는 보너스지 강제가 아님

**사전 테스트:** 10명에게 Quick Add만 주고 작업 5개 생성 시키기. 파싱 정확도 90% 미만이면 패턴 조정.

---

### Death Cause 4: "Windows 데스크톱 앱은 아무도 안 깐다"

**살인 동기:** 2026년에 데스크톱 앱을 설치하는 건 진입장벽이 높다.
SmartScreen 경고까지 뜨면 대부분 포기.
웹앱이었으면 URL 하나로 끝나는데.

**확률:** 중간 (30%)

**예방 조치:**
- Vite 빌드(`npm run build`)로 **PWA 버전 병행 배포** — 설치 없이 브라우저에서 사용
- 데스크톱은 "프리미엄 경험" 포지셔닝 (오프라인, 네이티브 알림)
- 코드 사이닝 우선 처리 — SmartScreen 경고 제거
- 랜딩페이지에 "Try in Browser" 버튼 우선 배치

**사전 테스트:** 랜딩 → 설치 전환율 측정. 5% 미만이면 PWA를 메인으로 전환.

---

### Death Cause 5: "혼자 만들어서 속도가 안 나온다"

**살인 동기:** 1인 개발자가 데스크톱 + 모바일 + 랜딩 + 마케팅을 다 하려다
핵심 기능 완성 전에 지쳐서 중단.

**확률:** 높음 (35%)

**예방 조치:**
- **Week 1에 [1]+[2]만 구현** — 편차 표시 + Daily Pulse가 MVP의 전부
- 모바일은 PMF 확인 전 절대 착수하지 않음
- 랜딩페이지는 Notion/Carrd 등 노코드로 대체
- AI 추천 고도화는 뒤로 미룸 — 현재 heuristic-recommender로 충분
- 마케팅은 글 1개 + 커뮤니티 공유 수준

**사전 테스트:** 매주 "이번 주 핵심 1가지"만 정하고 그것만 완료. 2가지 이상 동시 진행 금지.

---

### Death Cause 6: "데이터가 쌓여야 가치가 보이는데, 그때까지 기다릴 사람이 없다"

**살인 동기:** 주간 정확도 추이, 패턴 감지 등 핵심 가치가 2~4주 데이터 축적 후에야 작동.
Cold start problem.

**확률:** 중간 (25%)

**예방 조치:**
- 즉시 편차 표시는 **첫 완료부터** 작동 — cold start 없음
- Daily Pulse는 **하루 1개 완료부터** 의미 있는 카드 표시
- 주간 추이는 **3일부터** 미니 차트 생성 (7일 안 기다림)
- 샘플 데이터로 "이렇게 보입니다" 프리뷰 제공

### Top 3 Kill Decisions (이걸 안 하면 죽는다)

1. **첫 작업 완료 시 즉시 편차 표시** — 차별화의 첫 인상
2. **편차를 비난이 아닌 개선 추이로 프레이밍** — 스트레스 방지
3. **Week 1에 [1]+[2]만 집중, 나머지 전부 뒤로** — 번아웃 방지
