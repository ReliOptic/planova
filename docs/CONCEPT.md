# Planova — Product Concept

> "시간이 어디로 새는지 보여주는 도구"

## Positioning

대형 경쟁사(Google Calendar, Todoist, Notion, ClickUp)는 전부 **"할 일을 관리"** 하는 도구.
Planova의 니치는 **"계획과 실제의 차이를 줄이는"** 도구.

```
         많은 기능
            |
   Notion   |   ClickUp
            |
   ---------+---------→ 팀용
            |
   개인용   |
            |
  Todoist   |   Asana
            |
         적은 기능

Planova: 개인용 + 적은 기능 + "시간 편차 측정"이라는 고유 축
```

## Core Philosophy

Toyota Way의 7대 낭비(무다)를 시간관리에 적용하되, **제품에 Toyota/린/무다라는 용어는 절대 노출하지 않는다.**
사용자 언어로 번역:

| 제조업 낭비 | 시간관리 등가물 | 사용자에게 보이는 말 |
|------------|----------------|---------------------|
| 과잉생산 | 안 해도 될 일을 함 | "백로그에 2주 넘게 머문 작업" |
| 대기 | 작업 사이 빈 시간 | "빈 시간 1h 15m" |
| 운반 | 컨텍스트 스위칭 | "오늘 그룹 변경 7회" |
| 과잉가공 | 예상보다 오래 걸림 | "+23분 초과" |
| 재고 | 백로그 비대화 | "대기 중 작업 15개" |
| 동작 | 일정 재배치 반복 | "오늘 블록 이동 12회" |
| 불량 | 작업 재시도 | "같은 작업 3회 반복 생성" |

## Differentiator: Plan-vs-Actual Gap

기존 코드에 이미 존재하는 구조:
- `Task.durationMinutes` — 계획 시간
- `ScheduleBlock.(endTime - startTime)` → `blockDurationMinutes` — 실제 시간
- `TaskViewModel`에서 두 값 모두 접근 가능

**이 차이를 시각화하는 생산성 도구는 현재 시장에 없다.** (OEE 대시보드는 제조업에만 존재)

## Target User

**프리랜서, 1인 사업자, 재택근무자**
- 관리자가 없어서 자기 시간을 스스로 감시해야 하는 사람들
- 투두 + 캘린더 정도 쓰지만, 피드백 루프 없이 반복만 하는 사람들
- "열심히 하는 것 같은데 시간이 어디로 가는지 모르겠다"는 느낌

## Message Frame

| 하지 않을 것 | 할 것 |
|-------------|------|
| "생산성 도구" | "시간이 어디로 갔는지 보여주는 도구" |
| "일정 관리" | "계획과 실제의 차이를 줄이는 도구" |
| "AI가 추천" | "당신의 패턴을 당신에게 보여줌" |
| "효율적인 시간 관리" | "낭비되는 시간을 찾아드립니다" |

## Technical Moat

- 100% 오프라인 — 데이터가 기기를 떠나지 않음
- 로그인 없음 — 설치 즉시 사용
- IndexedDB 로컬 저장 — 서버 비용 0
- 선택적 AI (OpenRouter) — 유료 API는 사용자 선택

## Strategic Path

```
Phase 1: 데스크톱 PMF (Windows)
  → 핵심 편차 피드백 루프 구현
  → 한국어 타겟, 프리랜서 커뮤니티 배포

Phase 2: PMF 검증
  → 7일 리텐션 측정 (주간 리포트 2주 연속 확인율)
  → 메시지 A/B 테스트

Phase 3: 모바일 Companion
  → 풀 앱 아님, 읽기 + 체크 위주
  → 데스크톱이 메인, 모바일은 리포트 확인용
```
