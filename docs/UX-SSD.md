# Planova — User Experience & Service Scenario Design (SSD)

**Version:** 0.1
**Date:** 2026-04-13

---

## 1. Design Principles

### 1.1 Core UX Philosophy: "Zero-Friction Deviation Loop"

```
사용자가 이미 하는 행동에서 데이터를 추출하고, 즉시 피드백을 준다.
추가 입력을 요구하지 않는다.
```

### 1.2 Three Rules

1. **입력은 1줄 이하** — 2 액션 이상 요구하면 이탈한다
2. **피드백은 즉시** — 4주 후가 아니라 완료 직후 2초 이내
3. **숫자는 비난이 아닌 추이** — "+23분"은 나쁜 게 아니라 "다음엔 예측 조정" 신호

### 2.1 언어 일관성 이슈

현재 UI 영/한 혼용:
- 영어: "Save Task", "New Task", "Duration", "Priority", "No pending tasks"
- 한국어: "반복 일정", "모달 닫기", "추천 활성화까지"

**결정:** 타겟이 한국 사용자이므로 **전체 한국어화** 진행. 단, 기술 용어(Backlog 등)는 영어 유지 가능.

## 3. Service Scenario Design (SSD)

### 3.1 Scenario A: 아침 — 하루 계획

```
시간: 08:55, 김프리가 PC 앞에 앉음
상태: Planova 열림, 오늘 타임라인 표시

┌─────────────────────────────────────────────────┐
│  타임라인                                        │
│                                                  │
│  09:00 │ ░░ 이메일 확인 30m ░░░ │  ← Ghost       │
│  09:30 │ ░░░░░░░░░░░░░░░░░░░░░ │    (어제 했음)  │
│  10:00 │ ░░ 디자인 작업 2h ░░░ │  ← Ghost       │
│  10:30 │ ░░░░░░░░░░░░░░░░░░░░░ │                │
│  11:00 │ ░░░░░░░░░░░░░░░░░░░░░ │                │
│  11:30 │ ░░░░░░░░░░░░░░░░░░░░░ │                │
│  12:00 │                       │                │
│                                                  │
│  김프리의 행동:                                   │
│  1. "이메일 확인" ghost 클릭 → 오늘 블록으로 변환  │
│  2. "디자인 작업" ghost 클릭 → 변환               │
│  3. 14:00 빈 칸 클릭 → "클라이언트 미팅 1h" 입력  │
│                                                  │
│  총 입력: 클릭 2회 + 텍스트 1줄                   │
│  결과: 하루 계획 완성                             │
└─────────────────────────────────────────────────┘
```

**마찰 분석:**
- 현재: 모달 3회 열기 × 9필드 = 27 입력
- 변경: 클릭 2회 + 1줄 = 3 입력
- **감소: 89%**

### 3.2 Scenario B: 오전 — 작업 실행 & 완료

```
시간: 09:00~09:38
상태: "이메일 확인" 블록 진행 중 (계획 30분)

┌──────────────────────────────────────────┐
│  09:00 │ ███████████████████████ │        │
│        │  이메일 확인            │        │
│        │  30m                    │        │
│  09:30 │ ████████ ← 현재 시간   │        │
│        │                        │        │
│  09:38: 김프리가 "완료" 클릭                │
│                                          │
│  ┌────────────────────────────┐           │
│  │  이메일 확인    ✓ 완료     │           │
│  │  계획: 30m → 실제: 38m    │           │
│  │  ┌──────────┐              │           │
│  │  │ +8분 ▲   │  (주황)     │           │
│  │  └──────────┘              │           │
│  │                            │           │
│  │  블록이 아래로 살짝 늘어남    │           │
│  │  (8분만큼 시각적 확장)       │           │
│  └────────────────────────────┘           │
│                                          │
│  김프리의 반응: "8분 초과했네. 내일은      │
│  이메일 확인을 40분으로 잡아야겠다"         │
│                                          │
│  → 이것이 피드백 루프의 핵심 순간          │
└──────────────────────────────────────────┘
```

**Deviation Feedback Timing:**
- 완료 클릭 → 편차 표시: < 100ms (로컬 계산)
- BJ Fogg: "행동 직후 2초 이내 보상" → 충족

### 3.3 Scenario C: 저녁 — Daily Pulse

```
시간: 18:00 (workHours 종료)
상태: 오늘 5개 작업 완료

┌────────────────────────────────────────────┐
│                                            │
│  ┌──────────────────────────────────┐      │
│  │  오늘의 시간 펄스                  │      │
│  │                                    │      │
│  │  ■■■■■■■■□□  정확도 82%           │      │
│  │                                    │      │
│  │  계획 5h 30m  →  실제 6h 12m      │      │
│  │  총 편차 +42분                     │      │
│  │                                    │      │
│  │  ─ 시간대별 밀도 ──────────────    │      │
│  │  09 10 11 12 13 14 15 16 17 18    │      │
│  │  ██ ██ ██ ░░ ██ ██ ░░ ██ ██      │      │
│  │              ↑        ↑            │      │
│  │            점심    빈 시간 45m     │      │
│  │                                    │      │
│  │  가장 초과: 디자인 작업 +28분 ▲    │      │
│  │  가장 정확: 클라이언트 미팅 +2분   │      │
│  │                                    │      │
│  │  ─ 이번 주 추이 ──────────────    │      │
│  │  월 71% → 화 78% → 수 82% ▲      │      │
│  │                                    │      │
│  │  3일 연속 기록 중                   │      │
│  │  (내일 빠져도 유지돼요)             │      │
│  │                                    │      │
│  │         [ 닫기 ]                   │      │
│  └──────────────────────────────────┘      │
│                                            │
│  김프리의 반응:                              │
│  "디자인 작업이 항상 초과하네...               │
│   내일부터 2h30m으로 잡아야지"                │
│                                            │
│  → 자기교정 사이클 시작                      │
└────────────────────────────────────────────┘
```

**Pulse 표시 조건:**
- 최소 1개 완료 작업 → 기본 카드 (정확도 + 총 편차만)
- 2개 이상 → 빈 시간, 최대/최소 편차 추가
- 3일 이상 → 주간 추이 추가

## 5. Emotional Design Map

사용자가 각 단계에서 느끼는 감정과 의도한 감정:

| 순간 | 현재 감정 | 목표 감정 | 설계 수단 |
|------|----------|----------|----------|
| 앱 열기 (빈 타임라인) | "또 처음부터..." 피로 | "어제꺼 쓸까" 편안함 | Yesterday Ghost |
| 작업 입력 | "귀찮다" 저항 | "바로 되네" 속도감 | 1-line Quick Add |
| 작업 완료 | 무감각 (아무 일 없음) | "오 좀 초과했네" 호기심 | Instant Deviation |
| 하루 끝 | "오늘 뭐 했지?" 불안 | "정확도 82%, 어제보다 나아졌네" 성취 | Daily Pulse |
| 2주차 | "이거 계속 해야 하나..." | "패턴 보이니까 의미 있네" 발견 | 패턴 추천 |
| 하루 빠짐 | "망했다, 포기" 죄책감 | "괜찮아, streak 유지됨" 안도 | Streak Freeze |

## 6. Micro-Interaction Specifications

### 6.1 Deviation Badge Animation

```
완료 클릭 시:
  0ms    — 체크 아이콘 등장 (scale 0 → 1, spring)
  200ms  — 블록 색상 전환 (편차 색상으로 fade, 300ms)
  400ms  — 편차 배지 등장 (slide-up + fade-in, 200ms)
  600ms  — 블록 높이 조정 (실제 시간 반영, spring 300ms)
```

### 6.2 Daily Pulse 등장

```
workHours 종료 시:
  0ms    — 하단에서 카드 slide-up (400ms, ease-out)
  400ms  — 정확도 게이지 fill 애니메이션 (800ms, ease-in-out)
  1200ms — 숫자 카운트업 (400ms)
```

### 6.3 Ghost 상호작용

```
호버:
  - opacity 30% → 60% (200ms)
  - 점선 border → 실선 (200ms)
  - "클릭하여 오늘 일정에 추가" 툴팁

클릭:
  - Ghost → 실제 블록 모핑 (spring, 300ms)
  - opacity 60% → 100%
  - 색상 활성화
```
